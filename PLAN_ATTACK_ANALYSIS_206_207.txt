## 206. attackanalysis.py (módulo novo)

**Arquivo:** `src/mytools/web/attackanalysis.py`
**Teste:** `tests/web/test_attackanalysis.py`
**Entry point:** `mytools-analysis = "mytools.web.attackanalysis:main"`

### Técnicas
- `attack_path` — Gerar grafo de caminhos de ataque (networkx.DiGraph + matplotlib)
- `exploit_suggest` — Consolidar exploits de todos os findings

### Funcionalidades
- Recebe JSON de findings (output de attackaudit ou outros módulos)
- Conecta findings relacionados em grafo de dependências
- Renderiza PNG/SVG via matplotlib
- Exporta DOT format (graphviz)
- Cores por severidade: critical=vermelho, high=laranja, medium=amarelo, low=azul, info=cinza

### Exemplo de attack path
```
XSS Reflected → Cookie Theft → Session Hijack → Admin Access → Data Exfil
```

## Notas

- Campo `exploit: str = ""` tem default vazio (não quebra testes existentes)
- Módulos info-only (certcheck, dnssec, darkweb, emailbreach) recebem `exploit: str = ""` sem popula-lo
- Cada exploit deve ser um comando/string copiável
- Exploits usam placeholder `<TARGET>` (usuário substitui pelo alvo)
- Módulos DNS usam `<DOMAIN>` e `<NAMESERVER>`
- Coluna `Ferramenta` indica ferramenta externa recomendada (metasploit, hydra, XSStrike, wfuzz, etc.)
- Ferramentas listadas são apenas referência; o exploit em si continua sendo o comando/payload
