# MyTools Roadmap v2 — Deduplicado
# ~180 itens únicos, organizados por categoria

## 1. Recon & Discovery

1. Passive Subdomain Enumeration	crt.sh, VirusTotal, SecurityTrails, Shodan
2. Subdomain Takeover Check	Verificar CNAMEs apontando para serviços não reclamados (GitHub Pages, Heroku, S3)
3. DNS History	Histórico de registros DNS (mudanças de IP, nameservers)
4. WHOIS History	Histórico de mudanças de ownership do domínio
5. IP ASN/ISP Info	Enriquecer IPs com dados BGP (ASN, organização, país)
6. Technology Fingerprint	Versões exatas via headers, meta tags, scripts
7. Google Dorking	Gerar dorks e coletar resultados via scraping
8. Email Breach Check	Verificar se emails do alvo apareceram em vazamentos (HaveIBeenPwned)
9. Social Engineering Recon	Coletar emails, nomes, cargos de funcionários do alvo
10. Paste/Leak Monitoring	Monitorar pastebin, GitHub, gitlab por credenciais do alvo
11. Dark Web Monitoring	Buscar menções do domínio em forums underground
12. OpenAPI/Swagger Discovery	Buscar spec exposta em /swagger.json, /openapi.yaml
13. GraphQL Playground Exposure	Detectar GraphQL Playground/GraphiQL exposto
14. Source Map Discovery	Buscar .map files de JavaScript (revelam código fonte)
15. Version Control Leak	Detectar .git, .svn, .hg expostos
16. Config File Detection	Buscar .env, config.json, settings.py, web.config
17. Backup File Detection	Buscar backups (.bak, .old, .save, ~)

## 2. Web Security Core

18. Open Redirect	Detectar redirecionamentos abusáveis (/redirect?url=http://evil.com)
19. CRLF Injection	Injeção de headers via \r\n; inclui HTTP Request Splitting via CRLF
20. SSTI (Server-Side Template Injection)	Testar Jinja2, Twig, Freemarker, etc. via bodies, headers, HTTP methods
21. SSRF básico	Detectar endpoints que fazem requests internos (via parâmetros com URL)
22. XXE Detection	Testar uploads de XML com entidades externas (SVG, DOCX/XLSX, SOAP)
23. NoSQL Injection	Payloads para MongoDB/Redis/CouchDB ({"$gt":""}, {"$ne":null})
24. LDAP Injection	Injeção em filtros LDAP via parâmetros de busca
25. XPath Injection	Injeção em consultas XPath
26. Server-Side Includes (SSI)	Testar injeção de SSI em servidores Apache/Nginx
27. Prototype Pollution	Testar parâmetros de URL que afetam objetos JS internos
28. Deserialization (PHP/Java/Python)	Testar endpoints que recebem objetos serializados
29. Cache Poisoning	Detectar cache key poisoning via headers não-normalizados
30. Web Cache Deception	Paths que o cache armazena mas a app não deveria cache
31. HTTP Method Override	X-HTTP-Method-Override, _method para bypass de ACL
32. HTTP Parameter Pollution	Testar HPP em diferentes positions (query, body, headers)
33. Blind XSS via callback	Injetar payloads que disparam webhook quando renderizados em painéis internos
34. CORS Misconfiguration profundo	Testar null origin, subdomain wildcard, credenciais com *
35. Clickjacking via Embedded Frames	Testar se X-Frame-Options/CSP pode ser bypassado
36. Host Header Injection	Testar injeção via Host header em responses
37. Header Injection via URL params	Injetar headers via parâmetros que aparecem em redirect/location
38. Log Injection	Injetar conteúdo em logs via User-Agent, headers customizados
39. Log4Shell via HTTP Headers	Testar JNDI injection em headers customizados

42. CSS Injection via Style	Injetar CSS para extrair dados (ex: background: url(...))

45. XSS via <details open ontoggle>	XSS sem interação do usuário
46. XSS via <img src=x onerror>	Vetores clássicos mas eficazes
47. XSS via <video> <source>	XSS via eventos de mídia
48. XSS via <link> <a> CSS	XSS via CSS import
49. XSS via <base> Tag	Manipular base URL para redirecionar requests
50. XSS via javascript: URI	Navegação via javascript: em diferentes contextos
51. XSS via data: URI	Injetar data:text/html em diferentes contextos
52. XSS via <iframe srcdoc>	XSS via iframe srcdoc

54. XSS via Custom Elements	XSS via web components customizados
55. XSS via Shadow DOM	XSS via shadow roots

90. Malformed HTTP Version	HTTP/1.3, HTTP/2.0, HTTP/9.9 para testar parsing
91. Null Request Byte	Injetar byte null no início do request para confundir parser
92. Whitespace in Header Names	Host : a.com com espaço extra

94. Absolute URI in Request	GET http://b.com/path HTTP/1.1 para testar proxy behavior
